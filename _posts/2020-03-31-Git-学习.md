## Git 是什么？

### 直接记录快照，而非差异比较

- 基于差异的版本控制
- Git 基于快照流的方式存储数据：
  - 每当提交更新或保存项目状态时，基本就会对当时全部文件创建一个快照并保存这个快照索引；
  - 为了效率，没有修改的文件，Git 不再重新存储该文件，而是保留一个链接指向之前存储的文件。

### Git 保证完整性

采用校验和机制，数据存储前先做检验和。计算校验和的机制叫做 SHA-1 散列。基于文件的内容或目录结构计算出来，是一个由 40 个十六进制字符组成的字符串。
